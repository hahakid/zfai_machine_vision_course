# 在以往的教材和图像处理中，课程开始以图像处理的计算和函数进行理论介绍.
# 图像处理虽然贴近人的视觉感知（占人五感输入的绝大部分），在理解上其实比其它类型的数据更直观和友好。
# 但是采用的例程图像均来自精心的挑选，也就是专用性较强，适合理论操作和学习。
# 近年发展的深度学习，推崇算法的通用性、泛化适用性。也就是卷积特征在一系列任务上均能够取得更加显著的结果。
# 另一方面，我个人早期图像研究也是从直接的工业相机入手，构建交通信号灯模型。实际数据采集，特别是在开放场景，会对图像数据带来不同哦哦那个类型的噪声。
# 这些来自镜头、畸变、环境光照、拍摄参数等问题带来的噪声会影响采样目标，以及后续算法。如何与实际应用打交道，实际上更具有挑战性，但也并非高不可攀。
# 与其在挑选的图像上，成功的运行实例函数，不如直接面对环境数据，开展实践性学习更有操控性和第一体验感。
# 因此，在初期的课程上，设计了直接对传感器realsense的调试，希望通过若干例程介绍如何通过一个传感器来获取环境中的图像信息，并存储下来。
# 后续课程虽然在图像处理中也采用“挑选的”图像数据，更鼓励对现实采集图像进行尝试，验证相关函数，参数的适用性。
# 同时，在涉及深度学习，数据驱动的研究中，将以个人以往研究经验。设计包含： 传感器调试（选型）--数据采集--数据标注--样本均衡性--建模--量化评估
# 在内的全流程学习方案，而不仅仅是 download一个公开数据集，clone一个github的repo，然后就开始走官方的流程。
# 诚然，再进行特定理论研究的时候，我们希望问题能够更加聚焦，以使得我们能够将有限的精力集中在特定问题上，来保证研究的高效性。
# 但是，基于我个人的观察，但我们的研究目标/问题，仅仅来自文献；我们所需要研究的问题本身，或者至少大多数研究生，缺少对问题本身实用性的判断。
# 或者，我们如果一直围绕已有数据集存在的问题进行探索，是否会偏离实际需求。以及未经本地 “任务--场景--感知方案”上验证的算法模型，
# 我们是否能够把握或者寻找到真正存在的问题？这就引出了一个一直以来困扰我的哲学思辨，它来自一部虚构电影《奇怪国家的数学家》：
# 沿着错误的问题，无法找到正确的答案。 或者说： 沿着错误的问题寻找，是否能找到正确的答案？

# 引入相机传感器调试和数据采集，其目的是引入对实际环境、周边环境、伸手可及的环境视觉数据的最直观的感受。如果能够推动学生进行实际数据的采集，
# 在自己采集的数据上验证算法，以及发现算法失效，以及如何去优化，或者开发新的处理流程，应该比仅仅在给定数据上调试通过，必将引出有趣的效果。
# 当然，并不一定所有的学生都对图像处理具有浓厚的兴趣，就像摄影和ps，好的挑战总会吸引到喜欢它的人，这是一场双向奔赴。

# 本章节主要涉及opencv的图像级基础操作，由于图像为二维信息，大多数图像数组操作以块（k*k)
# 为基本的计算单元。
# 属于图像处理发展的古典主义时期（青铜时代？），不同的计算方式结合数学理论，以及算法设计者的智慧
# 被设计出来。
# 这一时期的图像处理方式五花八门，但在实际应用时需要通过了解具体函数的功能进行流程设计
# 并在此基础上进行参数的优化。
# 一旦设计流程包含的操作函数数量、先后顺序、参数规模变大，流程的调试和优化过程将十分复杂。
# 在部分的例程中，通过将操作函数嵌入到循环语句，并结合可视化，完成了部分人在环的调参测试。
# 这类算法的另一个局限在于“样本”制约，人的记忆认知相对有限，使得算法的调参过程低效，算法仅能够适应小规模数据，泛化能力弱。
# 另一类提升参数优化和参数泛化能力的方案则依赖：标注+统计分析。属于机器学习启蒙阶段，将在后续流程中介绍，
# 这一阶段的学习器，俗称learner，的学习能力往往较弱，属于低级智能，但能在相当规模的数据上获得泛化（在记忆认知上超过人）。
# 相比于更加智能化的 “ 通用卷积特征架构”， 经典图像处理操作显得低效、普适性差，但是现在的CNN
# 发展不是空中楼阁。他们的功能、设计思想、使用经验在现在依然具有活力，就像没有差的武功，只有平庸的武者（自己）。
